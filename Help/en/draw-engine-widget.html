Overview

The Draw Engine Widget is the main visual interface for viewing and editing your neural network model. It contains the component scheme visualization along with a list of available component classes. The widget provides two visualization modes: the legacy diagram view and the modern diagram view.

Accessing the Widget

The Draw Engine Widget is displayed in the main application window, typically in a tab or sub-window. It's automatically created when the application starts and is the primary interface for working with your model's structure.

Features

  • Dual Visualization Modes: Switch between legacy and modern diagram views
  • Component Classes List: Browse and search available component classes
  • Component Selection: Click components to select and view their properties
  • Navigation: Navigate into nested components by double-clicking
  • Scheme Updates: Automatic updates when components or connections change

Components

Component Classes List

The left panel displays a hierarchical list of available component classes. You can:

  • Browse component categories
  • Search for specific components
  • Double-click to add components to your model

Legacy Diagram View (UDrawEngineImageWidget)

The legacy diagram view provides a traditional schematic representation of your model. It shows components as boxes with connections between them.

Modern Diagram View (UModernDiagramWidget)

The modern diagram view provides an enhanced visual editor with improved interaction. For detailed information, see the Modern Diagram Widget documentation.

Usage

Adding Components

  1. Browse the Component Classes List on the left
  2. Find the component you want to add
  3. Double-click the component class to add it to your model
  4. The component will appear in the diagram view

Selecting Components

  • Single Click: Select a component to view its properties
  • Double Click: Navigate into a component (if it contains nested components)

Navigating the Model

  • Use the diagram view to see the hierarchical structure of your model
  • Double-click components to drill down into nested structures
  • Use the back button or context menu to navigate back up the hierarchy

Updating the Scheme

The diagram automatically updates when:

  • Components are added or removed
  • Connections are created or deleted
  • Component properties change (if they affect visualization)

Related Widgets
