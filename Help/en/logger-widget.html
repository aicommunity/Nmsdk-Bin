Overview

The Logger Widget displays log messages and system information from the NMSDK application. It provides real-time feedback about application events, errors, warnings, and debug information.

Accessing the Widget

The Logger Widget is displayed in a dock panel in the main application window. You can show or hide it using the Window → Logger menu item.

Features

  • Real-time Logging: Displays log messages as they occur
  • Multiple Log Levels: Shows information, warnings, errors, and debug messages
  • Automatic Updates: Continuously updates with new log entries
  • Scrollable View: Navigate through log history
  • Text Display: Plain text format for easy reading

Usage

Viewing Logs

The logger automatically displays messages from the application. Log entries include:

  • Timestamp information
  • Log level (Info, Warning, Error, Debug)
  • Message source
  • Message content

Log Levels

  • Info: General information messages
  • Warning: Warning messages about potential issues
  • Error: Error messages indicating problems
  • Debug: Detailed debug information (when debug mode is enabled)

Navigating Logs

  • Use the scrollbar to navigate through log history
  • The logger automatically scrolls to show the latest messages
  • You can scroll up to view older messages
Tip: The logger is useful for debugging issues and monitoring application behavior. Check it regularly to stay informed about system events.

Log Configuration

Log levels and output can be configured through:

  • The Status Panel - Enable/disable debug messages and detailed logging
  • Application settings
  • Command-line parameters

Related Widgets

  • Status Panel - Control logging options
  • Watch Window - Monitor component data
