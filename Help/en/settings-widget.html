Settings Widget

  • ← Back to Main Page

Overview

The Settings Widget provides a mechanism for arbitrary QObject-derived classes to receive load/save parameter events from the RDK framework. It acts as a bridge between the RDK parameter storage system and custom widgets that need to persist their settings.

Accessing the Widget

The Settings Widget is automatically created when the main application window is initialized. It operates in the background and doesn't have a visible interface.

Features

  • Parameter Persistence: Enables widgets to save and load their configuration automatically
  • Event Bridge: Connects RDK parameter storage events to custom widget handlers
  • Automatic Integration: Works seamlessly with the RDK framework's parameter management system

How It Works

The Settings Widget receives two types of events from the RDK framework:

  • LoadParameters: Triggered when the application loads saved parameters
  • SaveParameters: Triggered when the application saves current parameters

When these events occur, the widget emits corresponding signals that can be connected to custom widget functions, allowing them to save or restore their state.

Usage

This widget is primarily used internally by the framework. Application developers typically don't interact with it directly, but it ensures that all widgets can properly save and restore their settings when projects are saved or loaded.

Note: The Settings Widget is a system component that works automatically. You don't need to configure it manually.
