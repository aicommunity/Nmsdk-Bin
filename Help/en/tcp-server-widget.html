Overview

The TCP Server Widget allows you to control and monitor a TCP server that can communicate with your neural network model. This enables external applications to interact with your model over a network connection.

Accessing the Widget

The TCP Server Widget can be opened from the Window → TCP Server menu item. It appears in a separate window or dock panel.

Features

  • Server Control: Start and stop the TCP server
  • Configuration: Configure server settings (port, address, etc.)
  • Status Monitoring: View server status and connection information
  • Auto-restart: Automatic server restart on errors
  • Network Communication: Enable external applications to communicate with your model

Usage

Starting the Server

  1. Open the TCP Server Widget
  2. Configure server settings (if needed)
  3. Click Start Server
  4. The server will start listening for connections

Stopping the Server

  • Click Stop Server to stop the server
  • All active connections will be closed

Configuring Server Settings

Configure the server before starting:

  • Set the server port number
  • Configure network address (if needed)
  • Set other server parameters
  • Click Apply to save settings

Resetting the Server

Use the Reset button to reset server state and clear any errors.

Server Status

The widget displays:

  • Current server state (running/stopped)
  • Number of active connections
  • Server port and address
  • Error messages (if any)

Auto-restart Feature

The server can be configured to automatically restart if it encounters errors or stops unexpectedly. This helps maintain connectivity for external applications.

Warning: Make sure the server port is not already in use by another application. Using a port that's already in use will prevent the server from starting.

Network Communication

Once the server is running, external applications can:

  • Connect to the server using the configured port
  • Send commands to control the model
  • Receive data from the model
  • Monitor model state in real-time

Related Widgets

  • Status Panel - Overall system status
  • Logger Widget - View server log messages
