Status Panel

  • ← Back to Main Page

Overview

The Status Panel displays real-time information about the current state of your model execution, including time, step count, frames per second (FPS), and global project settings. It's located at the bottom of the main application window.

Accessing the Widget

The Status Panel is always visible at the bottom of the main application window. It cannot be hidden but can be resized.

Features

  • Execution Statistics: Displays current time, step number, and FPS
  • Global Settings: Checkboxes for global project configuration options
  • Thread Mode: Switch between single-threaded and multi-threaded execution
  • Auto-update Controls: Enable/disable automatic property updates
  • Logging Options: Control debug messages and detailed logging

Displayed Information

Execution Statistics

  • Time: Current simulation time
  • Step: Current calculation step number
  • FPS: Frames per second - calculation rate

Global Settings Checkboxes

  • Show Debug Messages: Enable/disable debug message display in the logger
  • Detailed Log: Enable detailed logging mode
  • Auto-update Properties: Automatically update component properties during execution
  • Auto-save States: Automatically save component states
  • Auto-save Project: Automatically save the project at regular intervals

Usage

Monitoring Execution

The Status Panel continuously updates during model execution, showing:

  • How long the simulation has been running
  • How many calculation steps have been completed
  • The current calculation speed (FPS)

Changing Global Settings

Click the checkboxes to enable or disable global settings:

  • Changes take effect immediately
  • Some settings (like Thread Mode) trigger a configuration save
  • Settings are persisted in the project configuration

Thread Mode

The Thread Mode setting controls how the model executes:

  • Single-threaded: Sequential execution (slower but more predictable)
  • Multi-threaded: Parallel execution (faster but requires thread-safe components)

Changing thread mode triggers an automatic configuration save.

Warning: Changing thread mode during execution may cause instability. It's recommended to pause or stop execution before changing this setting.

Property Update Interval

The Status Panel can control the interval at which component properties are automatically updated. This helps balance between real-time feedback and performance.

Related Widgets

  • Logger Widget - View log messages (affected by debug settings)
  • Calculation Channels Widget - Manage calculation channels
  • Watch Window - Monitor component data
