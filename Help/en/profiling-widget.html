Profiling Widget

  • ← Back to Main Page

Overview

The Profiling Widget displays performance information about component execution times in your neural network model. It helps you identify bottlenecks and optimize model performance by showing how much time each component takes to execute.

Accessing the Widget

The Profiling Widget can be opened from the Window → Profiling menu item. It appears in a dock panel showing a table of execution times.

Features

  • Execution Time Display: Shows execution time for individual components
  • Component Selection: Add specific components to the profiling table
  • Group Components: Add all sub-components of a selected component
  • Base Components: Always displays base components (Model, Overhead, Full Step, Gui)
  • Real-time Updates: Execution times update during model execution
  • Performance Analysis: Identify slow components and optimization opportunities

Usage

Viewing Base Components

The profiling table always displays base performance metrics:

  • Model: Total model execution time
  • Overhead: System overhead time
  • Full Step: Complete calculation step time
  • Gui: GUI update time

Adding Components to Profile

  1. Click Select Component
  2. Navigate to the component you want to profile
  3. The component will be added to the profiling table
  4. Execution time for this component will be displayed during model execution

Adding Component Groups

  1. Click Add Group Components
  2. Select a component that contains sub-components
  3. All sub-components will be added to the profiling table
  4. This is useful for analyzing hierarchical component structures

Adding All Components

Click Select All Components to add all external (non-nested) components to the profiling table.

Removing Components

  • Click Delete All to remove all non-base components from the table
  • Base components (Model, Overhead, Full Step, Gui) cannot be removed

Interpreting Results

The profiling table shows:

  • Component name
  • Execution time per step (in milliseconds or microseconds)
  • Percentage of total execution time

Use this information to:

  • Identify components that take the most time
  • Find optimization opportunities
  • Balance computational load
  • Verify that components are executing as expected
Tip: Focus on components with high execution times. These are the best candidates for optimization.

Performance Optimization

When analyzing profiling data:

  1. Look for components with unexpectedly high execution times
  2. Check if component groups show uneven distribution of computation
  3. Consider optimizing or replacing slow components
  4. Verify that overhead is reasonable compared to model execution time

Related Widgets

  • Status Panel - Overall execution statistics
  • Graph Widget - Visualize performance trends
  • Watch Window - Monitor component behavior
